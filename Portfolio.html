Contributions Type:

Group Project(Personal Contribution Hardware and Software Labs)

Figure 5: Check adapter status again and the wireless adapter should now be in monitor mode

Figure 6: Using the command "sudo airodump-ng wlan0mon" the adapter is now capturing the communication packets between various clients and various routers

Figure 8: Simultaneously, aireplay-ng sends management frames to clients to force them to disconnect and capture the handshake when they attempt to reconnect to the known wireless network.

Figure 9: Once Clients attempt to reconnect we get the two indicators of the handshake being captured that is the Eapol Appearing in notes section for the specific client attempting to connect and at the top left , the message WPA handshake: (mac address of the target)

Figure 10: At this stage the handshake is captured , thus here other tools and software are demonstrated to capture the handshake file using easier methods but following essentially the same process. These tools are the Raspberry PI Running Parrot OS using wifite , an ESP 32 Development board using deauth Firmware, Fern Wifi Cracker, Airgeddon or The Wifi Pineapple

Figure 11: Now that the capture file is saved usually in .cap or .pcap file extension, opening Wireshark and filtering traffic by EAPOL protocol to identify handshake packets with the best case scenario having captured the full handshake but partial handshakes may still be cracked.

Figure 12: Using a known password list against the capture file, if the password is in the capture file,

Figure 13: The goal for the user is to set a password not in the attacker's password list so that when the password list has reached the last entry and fails to match the password in the capture file, it returns the message KEY NOT FOUND

Figure 14: In the event the password is retrieved and the attacker can connect to the wireless network, they can now perform data sniffing using Wireshark and reassemble tcp (Transmission Control Protocol) / udp (User Datagram Protocol) streams or export objects meaning data communicated in any insecure protocol such as HTTP (Hypertext Transfer Protocol) or Telnet can be intercepted and reconstructed.
